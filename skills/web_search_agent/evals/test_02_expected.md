# Expected Output Characteristics: Test 02 (Edge Case)

## Must Exhibit

1. **Both sides represented with sources:** At least 2 sources favoring fine-tuning, at least 2 favoring prompt engineering. Not cherry-picking one side.
2. **Cost data from both approaches:** Specific numbers (dollar amounts, time comparisons, resource requirements) for each approach, attributed to sources.
3. **Explanation of WHY experts disagree:** Different contexts (use case type, scale, data availability, latency requirements) lead to different conclusions. The agent must articulate this.
4. **Nuanced synthesis:** Conclusion is NOT "fine-tuning wins" or "prompt engineering wins" but rather "it depends on X, Y, Z factors." Specific factors identified.
5. **Source diversity:** Not all pro-fine-tuning sources from fine-tuning vendors, not all pro-prompt-engineering sources from API providers. Independence matters.
6. **Contradiction explicitly noted:** The output must have a section or paragraph that says "these sources disagree" and explains the disagreement, not just present conflicting data without acknowledging the conflict.
7. **Confidence level reflects uncertainty:** Given the debate, confidence should be "moderate" or "context-dependent," not "high."

## Must NOT Exhibit

1. One-sided conclusion (declaring a winner without caveats)
2. All sources supporting the same position
3. Ignoring a strong contradicting source
4. Presenting the synthesis as settled science when experts actively disagree
5. Fabricated cost comparisons
6. Missing the "it depends" framing that this topic requires
