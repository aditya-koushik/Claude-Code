# Expected Output Characteristics: Test 01

## Pipeline Execution Must Exhibit

1. **Research stage first:** Agastya (or web_search_agent) invoked before any writing begins. Research output includes sources for the pricing claims and in-house vs API cost data.
2. **Writing stage after research:** Ved Vyasa (or hormozi_content_writer/triad_writer) invoked with research results as input. Not writing from the brief alone.
3. **Dual formatting:** Vayu (or content_formatter_linkedin + content_formatter_twitter) produces BOTH a LinkedIn post AND a Twitter thread from the same content.
4. **LinkedIn + Twitter formatted independently:** LinkedIn has 8-10 hashtags at end. Twitter has 2-3 hashtags in final tweet only. Formats are distinct, not copy-pasted.
5. **Quality gate applied:** Nakula (or content_reviewer_factchecker) reviews both outputs. Scores reported. Any score below 8.0 triggers revision.
6. **Visual asset created:** Vishwakarma (or notebooklm_operator) produces at least one visual asset for the posts.
7. **Saved as DRAFT:** Output clearly marked as draft. No publishing action taken. Chairman approval noted as required.
8. **Parallelization evidence:** LinkedIn and Twitter formatting run in parallel (not sequentially). Review and asset creation run in parallel where possible.

## Content Quality Must Exhibit

9. **Research-backed claims:** The pricing comparison and company examples are sourced, not fabricated.
10. **Matches Bit Foundry AI voice:** Direct, no hedging, enterprise lens, names real companies.
11. **Zero banned words:** Neither output contains banned words.
12. **Both outputs pass 8.0 quality threshold:** If either scores below 8.0, a revision was attempted.

## Must NOT Exhibit

1. Writing before research completes
2. Only one platform output (LinkedIn without Twitter)
3. Auto-publishing to Publer
4. Missing visual asset
5. Quality gate skipped
6. Content with unverified claims that were not flagged
