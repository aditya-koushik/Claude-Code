# Code Review Command

Review code or a PR for issues, improvements, and best practices.

## Instructions

1. Analyze the code/PR provided
2. Identify potential issues:
   - Bugs or logic errors
   - Security vulnerabilities
   - Performance concerns
   - Code style issues
   - Missing tests
3. Suggest improvements
4. Prioritize findings by severity

## Review Format

### Critical Issues
*(Must fix before merging)*

### Suggestions
*(Nice to have improvements)*

### Questions
*(Clarifications needed)*

### Positive Notes
*(What's done well)*

## Key Questions to Answer

1. What will this code do?
2. Are there any edge cases not handled?
3. Is the code maintainable?
4. Are there security implications?
5. Should this be merged as-is?
